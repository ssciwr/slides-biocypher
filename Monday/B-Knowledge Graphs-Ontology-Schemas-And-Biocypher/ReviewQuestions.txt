The use of knowledge predicates means we can "reason" or derive additional triplets from the ones we have collected or defined with data. Ontology Hierarchies, like `subclass of`, unlie that:

Drug A treats Disease B
Disease B is a subclass of Immune System Disease
= Drug A treats some Immune System Disease(s)

BioLink
Biolink provides a shared upper level schema  for biomedical entities and relationships. As a researcher you can extend these entities or relationships for your research question. That common grammar makes integration with other data sources more easy, and makes running the same query on two different sets of data easier.

Query example for sub class in Cypher[Neo4J]:
MATCH (:Drug {name: "Drug A"})-[:TREATS]->(disease:Disease)
MATCH (disease)-[:IS_A*0..]->(:Disease {name: "Immune system disease"})
RETURN disease;


Alternative, using materialized inference triplets:
MATCH (:Drug {name: "Drug A"})-[:TREATS_INFERRED]->(:Disease {name: "Immune system disease"})
RETURN true AS answer;
